### План статьи

**1. Введение: Зачем химии нужно машинное зрение?**

- **Актуальность:** Проблема «цифрового разрыва» между старыми печатными фондами и современными data-driven подходами. Зачем оцифровывать книги и формулы (поиск, сохранение знаний).

**2. Проблематика оцифровки химической литературы**

- **Что такое машинное зрение (CV) в контексте задачи:** 
  - что это такое + исторические справки 
  -  зачем оно нужно в рамках работы
  - Краткий переход от общих концепций CV к анализу документов (Document Image Analysis).

- **Специфика текста (OCR):** Артефакты старых сканов, большенство книг плохо отсканированы
- **Специфика структур (OCSR - Optical Chemical Structure Recognition):** 
  - Классификация химических формул: Какие в химии есть молекулы и реакции
  - Химическая формула — это граф, а не линейная последовательность токенов.
  - Проблема неоднозначности: одна молекула может порождать множество вариантов записи в формате SMILES.
  - Устаревшие нотации: особенности скелетной записи в старой литературе.

**Синтез:** Проблема совместного нахождения графики и текста на одной странице.

**3. Обзор существующих решений**

- Краткий анализ готовых инструментов (например, Tesseract для OCR, DECIMER / RxnScribe / MolScribe для OCSR).
- Чего им не хватает: почему «коробочные» решения плохо работают «из коробки» на ваших данных (старые сканы, русский язык) и почему нужна кастомная доработка.

**4. Архитектура решения:**

- **4.1. Обработка страницы целиком (Сегментация):** Машинное зрение (YOLO/Vision Transformers) против эвристики и первичной фильтрации. Разделение страницы на текстовые блоки и графические (формулы).
- **4.2. Текстовый поток (OCR):**
  - Применение графических фильтров (бинаризация, морфологические операции) перед OCR.
  - Извлечение текста и Prompt Engineering (использование LLM для исправления ошибок OCR и сшивки переносов).

- **4.3. Хемоинформационный поток (OCSR):**

  - Предобработка изображений формул (фильтры).
  - Распознавание молекул и алгоритмическая нормализация SMILES (например, через RDKit для приведения графов к каноничному виду).
  - *От молекул к реакциям:* Химическая классификация, распознавание стрелок и условий реакций. Обучение на примерах (Few-shot learning) и Prompt Engineering для связывания реагентов и продуктов.

**5. Интеграция данных и интеллектуальный поиск (RAG)**

- **Концепция Dual-Stream Retrieval (Двухпоточный поиск):** Как объединить поиск по тексту и поиск по структурам.
- **Поиск по формулам:** Как реализован поиск похожих структур внутри базы (молекулярные фингерпринты, векторные эмбеддинги графов).
- **Архитектура RAG:** Генерация ответов на естественном языке со строгими ссылками на первоисточник.

**6. Программная инженерия и конвейеризация (MLOps / Data Engineering)**

- Построение масштабируемого ETL-конвейера для потоковой обработки целых книг, а не отдельных страниц.
- Стандарты качества: API, соблюдение PEP-8, Unit-тестирование отдельных узлов конвейера (модулей фильтрации, сегментации, извлечения).

**7. Эксперименты и результаты**

- Сравнение различных графических фильтров и их влияние на качество (до/после).
- Оценка метрик (Precision/Recall для поиска, точность извлечения формул и реакций).
- Примеры успешной обработки сложных страниц (до/после).
